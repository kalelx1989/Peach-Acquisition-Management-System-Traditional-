# Peach-Acquisition-Management-System-Traditional-
## 1. 简介

本系统是一套专为传统桃经纪设计的全流程轻量化管理平台。涵盖了农户与客商自主注册、移动端现场收购录入（分拣与散装）、空框周转管理，以及PC端后台基础数据维护、财务结算、票据打印等功能。

### 核心子系统：
- **公共入口**：提供各端导航及人员自主注册功能。
- **移动端（员工现场使用）**：支持手机端便携操作，包括设备认证、扫码收购、空框管理。
- **PC后台端（管理/财务使用）**：数据大屏看板、人员管理、财务业务结算、报表及小票打印。

---

## 2. 角色与入口说明
系统的首页默认地址为：`http://您的域名/index.php`，在此页面可以选择进入相应的子系统。
系统包含以下主要角色：
1. **桃农/客商**：通过系统扫码或点击自主填写信息注册，获得专属**身份二维码**。
2. **现场员工（操作员）**：使用移动端设备，通过扫描农户/客商二维码进行快速业务录入。
3. **后台管理员**：使用PC端查看全盘数据、维护人员信息、核对交易及财务结算。

---

## 3. 桃农/客商自主注册
**访问路径**：网页访问 `http://您的域名/mobile_register.php` （通常系统会生成该页面的二维码供线下扫码进入）。
1. **选择身份**：进入后点击对应卡片：“我是桃农”或“我是客商”。
2. **填写信息**：
   - **桃农**：填写姓名（必填）、手机号（必填）、所属村庄。
   - **客商**：填写姓名（必填）、手机号（必填）、负责区域。
3. **获取二维码**：提交成功后，系统会自动为您生成专属的“身份二维码”并提供下载（请保存至手机相册）。
   > 💡 **温馨提示**：请妥善保存该二维码，在每次交易时出示给现场员工扫码，即可免去口头报信息的繁琐，实现极速身份关联录入。

## 4. 移动端操作指南（现场员工专用）
**访问路径**：由首页选择“移动端”进入，或直接访问 `http://您的域名/mobile/index.php`

### 4.1 设备安全认证
为了保障收储经营数据安全，系统实行**设备白名单机制**。初次在新手机或浏览器上访问移动端时：
- 页面中部会显示“您的设备标识符”（类似 `client_xxx_xxx` 的格式）。
- 需将此识别码告知后台管理员，管理员在后台绑定/授权该设备后，刷新页面方可展现核心收购功能菜单。

### 4.2 功能面板操作
认证成功后，移动端将展示以下三大核心功能：

#### 📦 功能一：分拣收购
用于详细登记不同品种、规格的精细化桃子收储数据。
- 点击“前往”进入。
- **身份录入**：扫描农户或客商提前准备好的二维码，系统将自动反显出身份信息。
- **填写数据**：录入桃子的品类、重量（毛重/皮重或净重）、件数、单价等。
- 系统会自动根据重量和单价计算总价，确认数据无误后点击保存，提交交易记录。

#### ⚖️ 功能二：散装收购
用于无需精细计件、直接打包盲称交易的散装业务，录单流程更加快捷。
- 核对并扫码确定农户或客商信息。
- 填写毛重、皮重等基础数据，快速完成整车或大批量的收购登记。

#### 🛒 功能三：空框管理
用于追踪收购站周转筐（塑料筐/纸箱等）的出借与归还转运动态。
- 扫码或选择对应的农户/客商。
- 录入发放（借出）或回收（归还）的框子数量。
- 随时掌握周转筐在各个人员手中的流转及积压情况，减少物资流失。

## 5. PC后台管理指南（管理员或财务）
**访问路径**：由首页选择“后台管理”进入，或直接访问 `http://您的域名/admin/login.php`

### 5.1 系统首页 (数据大屏 DashBoard)
系统管理员登录后默认进入首页，提供直观的宏观数据概览：
- **顶部数据卡片**：展示系统内当前累计的客商总数、累计农户总数、覆盖的桃品分类数量。
- **核心数据看板**：重点高亮聚合展示“今日交易总额”。
- **趋势分析图表**：以可视化折线图及柱状图的形式，直观呈现收购站近7日的交易额走势与交易重量（斤数）趋势，方便决策。
- **实时交易播报**：下方滚动展示系统内最新提交的每一笔收购交易流水、相关结算人员及详细金额。

### 5.2 基础数据管理
位于界面左侧主菜单栏。
- **客商管理 / 农户管理**：
  - 可以查看所有通过移动端自主注册或由管理员手动后台添加的人员名单。
  - 支持后台手动新增人员、编辑或修改错误信息。
  - 支持后台调取查阅或重新下载具体人员的身份二维码。

### 5.3 业务与财务结算管理
- **农户结算** / **客商结算**：
  - 基于现场员工在移动端提交的真实收购数据并同步至后台，财务人员可以在此模块进行账单核对。
  - 勾选满足条件的交易单进行批次打款或现金支付标记，维护实际真实的资金给付状态及剩余结欠状态。
- **操作回单（小票）管理**：
  - `回单维护`：管理与作废核销历史交易的错误回单记录。
  - `回单补打` / `自动打印`：电脑端外接热敏或针式打印机后，实现交易小票、入库单、结算收据回单的自动静默打印或手动二次补打，交还给送货司机或收购员，方便为现场业务提供可靠的纸质凭证交接留存。

### 5.4 系统运维设置
- **用户管理**：
  - 给公司内部人员增立、禁用或修改后台系统的管理员登录账号（如设立专职的主管/操作员账号等）。
  - 在此模块核心审批与管控移动端的收购操作权限（审核设备授权、管理新机器的认证 Client ID 等保障信息安全）。

## 6. 标准业务流转规范梳理
为了系统能在实际收购场景中高效平稳运作，建议按如下实操动线运行：
1. **开季入场准备**：让合作的农户、客商提前通过您的 `mobile_register.php` 短链接在手机端注册，并存下生成的个人专属身份二维码。同时现场收货员向管理员上报其手机设备指纹并获取收购端的录入权限。
2. **现场过磅交易**：农户开车到场过磅 ➜ 收购站员工用手机端扫农户二维码识别信息 ➜ 看货选品排等级定出单价 ➜ 员工当场在手机端打开“分拣收购”或“散装收购”填入果品类型、重量、单价并提交上传 ➜ 员工接着操作“空框管理”核实并发放车辆带走备用的空果筐。
3. **资金交割结算**：财务管理员坐镇PC端，进入“农户结算”，核实当日该名农户新产生的收购大单，通过微信或银行渠道完成转账（或付现），随后在系统内点击该笔交易单，标记结清款项。
4. **留档存根**：PC端自动或手动为刚才过账的交易/结算打出一式两联的单据，（站内留一联财务存根、另一联发派交由买方/卖方留底备查）。
